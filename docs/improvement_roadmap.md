# 개선 로드맵 (유튜버 리뷰 피드백)

> 외부 시각(주식 유튜버 리뷰)에서 나온 약점 → 차근차근 개선. 완료 시 `[x]`.

## [ ] 1. 검증된 트랙레코드가 얇다
- 문제: 시그널 자가 채점이 좋은 기능인데, **채점된 매수/매도 신호가 1건 수준**이라 "따라하면 돈 번다"를 증명 못 함.
- 방향(검토): **과거 데이터로 시그널 백필/백테스트** → 즉시 통계적으로 의미 있는 적중률 확보. (yfinance 히스토리로 과거 날짜의 시그널 재현 → 실제 N일 후 가격으로 채점)
- 보조: 적중률에 **표본 수(n) 같이 표기**(0% 오해 방지), 시그널 기록을 cron으로 매일 자동(앱 안 열어도 누락 X).

## [ ] 2. 데이터가 무료라 한계
- 문제: yfinance ~15분 지연, 가끔 값 튐(NVDA $600 사례 → 신문은 수정함). 정밀 트레이딩엔 부족.
- 방향: 시세 출처 이중화/검증(이상치 감지), CSV 현재가 자동 갱신(접속 시), 필요 시 유료 시세 옵션.

## [ ] 3. 패턴 분석은 'AI 추론'
- 문제: 헤드앤숄더·쌍바닥 등은 차트 이미지 인식이 아니라 LLM 추론(확신도 표기).
- 방향: 핵심 패턴 일부를 알고리즘으로 정밀 탐지(쌍바닥/추세선/삼각수렴), 백테스트로 패턴 신뢰도 측정.

## [ ] 4. 1인 운영 = 지속성·안정성 물음표
- 문제: 개발자 1명, 무료 서버(Oracle), 백업 전략 미흡.
- 방향: data/ 자동 백업(포트폴리오 CSV·매매일지 등), 서비스 헬스체크/자동 복구, 배포 안정화.

## [ ] 5. 기능이 많아 처음엔 복잡
- 문제: 탭 많아 입문자 학습곡선.
- 방향: 첫 진입 온보딩/요약 대시보드, 핵심 3~4개만 보이는 '간단 모드', 툴팁.
