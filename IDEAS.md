  #### 📌 #2 작업하며 아쉬웠던 점 / 기능 메모 (2026-06-28, 우선순위 미정)
  > 캐시 하네스를 만들며 눈에 띈 거친 부분 + 있으면 좋을 기능. 3순위 가기 전 보관용.

  **아쉬웠던 점(개선)**
  - **'지금 새로고침' UI 버튼 미연결** — `force_live=True` 파라미터(주가·뉴스)는 만들었는데
    실제 페이지에 새로고침 버튼이 아직 안 붙음. 캐시가 stale일 때 사용자가 강제로 최신화할 길 필요.
  - **stale 판정이 18시간 고정** — 거래일/주말/휴장 미고려. 주말엔 금요일 데이터가 stale로 떠서
    불필요한 라이브 호출 발생. '마지막 미장 마감 기준' freshness가 더 정확.
  - **캐시 미적용 경로 다수 잔존** — 현재 캐시 경유는 trade_signal·issue_tracker뿐.
    market_overview / portfolio_advisor / sector_scanner / quant_analyst / watchlist 종목추가검증
    등은 여전히 라이브 yfinance. 같은 `market_cache`로 점진 전환 필요.
  - **수집 실패 시 조용함** — collect_market_data가 실패/누락돼도 로그만 남고, 페이지는 그냥
    느려질 뿐(라이브 fallback). 인지 못 함 → #4 모니터링 하네스와 연결해 cron 실패 알림 필요.
  - **종목 추가 시 즉시 워밍 안 됨** — 관심/포트폴리오에 새 종목 넣으면 다음 새벽까지 캐시 없음
    → 첫 조회는 라이브(느림). 추가 시점에 그 종목만 즉시 캐시 워밍하면 매끄러움.
  - **FNG(CNN 공포탐욕)는 여전히 매번 라이브 HTTP** — 지수/주가는 캐시됐는데 시장국면의 FNG만
    캐시 밖. regime 데이터도 캐시에 포함하면 완전.

  **있으면 좋을 기능**
  - **뉴스 타임라인 활용** — 이제 종목별 뉴스가 누적되므로, "급등/급락한 날 무슨 뉴스가 있었나"를
    시그널·차트에 오버레이. 매매일지·데일리신문 근거로도 연결.
